│ Context                                                                                                                                                                                                                                           │
│                                                                                                                                                                                                                                                   │
│ Build a production-grade ingestion engine that discovers and ingests top developer profiles from both GitHub and HuggingFace into Postgres. Starting with top 5k users per platform (10k total), scaling to 5M.                                   │
│                                                                                                                                                                                                                                                   │
│ The existing octopod-backend has a basic GitHubClient (2 REST calls, no retry, single token) and HuggingFaceClient (4 REST calls, no retry). These need to be replaced with robust engines modeled after the reference implementation at          │
│ /Users/suniltiwari/Downloads/ingest_engines.                                                                                                                                                                                                      │
│                                                                                                                                                                                                                                                   │
│ Key requirements from user:                                                                                                                                                                                                                       │
│ - Tens of thousands+ scale with job queue, multi-token pool, resume from failure, progress tracking                                                                                                                                               │
│ - GitHub "top" = followers + stars (weighted rank normalization)                                                                                                                                                                                  │
│ - HuggingFace "top" = downloads + likes (weighted rank normalization)                                                                                                                                                                             │
│ - Both platforms simultaneously (5k GH + 5k HF)                                                                                                                                                                                                   │
│ - Separate tables: gh_users, gh_repositories, gh_commits, gh_activity_events, gh_checkpoints + hf_users, hf_models, hf_datasets, hf_checkpoints                                                                                                   │
│ - Parallel module structure: gh_ingest/ + hf_ingest/, minimal sharing                                                                                                                                                                             │

│ 1. Token Pool (app/ingest/gh/token_pool.py)                                                                                                                                                                                                       │
│                                                                                                                                                                                                                                                   │
│ Modeled directly from reference gh_ingest/token_pool.py:                                                                                                                                                                                          │
│ - TokenState dataclass: token, remaining (default 5000), reset_at, in_use                                                                                                                                                                         │
│ - TokenPool class with asyncio.Lock + asyncio.Condition                                                                                                                                                                                           │
│ - acquire(): blocks when all tokens exhausted, prefers token with most remaining budget                                                                                                                                                           │
│ - release(state, remaining, reset_at): updates budget from response headers, notifies waiting workers                                                                                                                                             │
│ - snapshot(): returns list of {remaining, reset_in_s} for monitoring                                                                                                                                                                              │
│                                                                                                                                                                                                                                                   │
│ 2. GitHub Client (app/ingest/gh/client.py)                                                                                                                                                                                                        │
│                                                                                                                                                                                                                                                   │
│ Rewrite of current app/service/clients/github_client.py. Key features:                                                                                                                                                                            │
│ - GraphQL bundling: Single query fetches user + repos + recent commits (reference USER_QUERY)                                                                                                                                                     │
│ - Token rotation: Uses TokenPool, updates budget from X-RateLimit-Remaining / X-RateLimit-Reset headers                                                                                                                                           │
│ - Retry with backoff: Exponential backoff with jitter (base * 2^attempt + random(0,1), max 60s)                                                                                                                                                   │
│ - Error classification: PermanentError (404/410/451 — skip user) vs TransientError (5xx/timeout — retry)                                                                                                                                          │
│ - Secondary rate limit handling: Respects Retry-After header on 403/429                                                                                                                                                                           │
│ - Shared httpx.AsyncClient with connection pooling (max_connections = concurrency * 2)                                                                                                                                                            │
│                                                                                                                                                                                                                                                   │
│ Public methods:                                                                                                                                                                                                                                   │
│ - fetch_user_bundle(login) → GraphQL: user + repos + commits in one round-trip                                                                                                                                                                    │
│ - fetch_user_events(login) → REST: public activity events                                                                                                                                                                                         │
│ - fetch_contributions(login) → GraphQL: contributionsCollection (PRs, issues, commits)                                                                                                                                                            │
│ - fetch_repo_commits_paginated(owner, name, max_commits, since) → GraphQL cursor pagination for backfill                                                                                                                                          │

│                                                                                                                                                                                                                                                   │
│ Rewrite of current app/service/clients/huggingface_client.py. Key features:                                                                                                                                                                       │
│ - Simple round-robin token rotation (HF rate limits are loose)                                                                                                                                                                                    │
│ - Retry with backoff: Same exponential backoff pattern                                                                                                                                                                                            │

│   d. Best-effort fetch_user_events() + upsert_events()                                                                                                                                                                                            │

│ 6. Discovery                                                                                                                                                                                                                                      │
│                                                                                                                                                                                                                                                   │
│ app/ingest/gh/discover.py — Stratified follower-band + star-based:                                                                                                                                                                                │
│ - 12 follower bands (50k+, 20k-50k, ..., 500-600) to work around GitHub's 1k search result cap                                                                                                                                                    │
│ - Separate pass: top repos by stars → aggregate owners' star counts                                                                                                                                                                               │
│ - Rank normalization: followers_rank + stars_rank combined with configurable alpha (default 0.5)                                                                                                                                                  │
│ - Returns List[UserCandidate] sorted by composite score                                                                                                                                                                                           │
│                                                                                                                                                                                                                                                   │
│ app/ingest/hf/discover.py — Top models by downloads + likes:                                                                                                                                                                                      │
│ - Phase 1: /api/models?sort=downloads — collect top 20k models                                                                                                                                                                                    │
│ - Phase 2: /api/models?sort=likes — collect top 10k models                                                                                                                                                                                        │
│ - Merge, aggregate by author, rank normalize (downloads_rank + likes_rank)                                                                                                                                                                        │
│ - Returns List[AuthorCandidate] sorted by composite score                                                                                                                                                                                         │

❯ how do i test it

⏺ Here's how to test the ingestion engine end-to-end:

  1. Start Postgres & Initialize Schemas

  # Start the database
  make docker-db

  # Create the ingestion tables
  make gh-init-schema
  make hf-init-schema

  2. Set API Tokens

  Create/update your .env file:

  # GitHub - get from https://github.com/settings/tokens (classic, no scopes needed for public data)
  GITHUB_TOKENS=ghp_token1,ghp_token2

  # HuggingFace (optional) - get from https://huggingface.co/settings/tokens
  HF_TOKENS=hf_token1

  More GitHub tokens = more throughput (each gets 5000 requests/hr).

  3. Quick Smoke Test (few users)

  # Ingest 2-3 specific GitHub users
  poetry run python -m app.ingest.cli gh-ingest --login torvalds --login gvanrossum --concurrency 2

  # Ingest 2-3 specific HF users
  poetry run python -m app.ingest.cli hf-ingest --user openai --user google --concurrency 2

  # Check status
  poetry run python -m app.ingest.cli status

  4. Discovery + Ingestion (top 100)

  # Discover top 100 GitHub users, save to file
  poetry run python -m app.ingest.cli gh-discover --top 100 --output /tmp/gh_logins.tsv

  # Ingest them
  poetry run python -m app.ingest.cli gh-ingest --input /tmp/gh_logins.tsv --concurrency 4

  # Same for HuggingFace
  poetry run python -m app.ingest.cli hf-discover --top 100 --output /tmp/hf_users.tsv
  poetry run python -m app.ingest.cli hf-ingest --input /tmp/hf_users.tsv --concurrency 4

  5. Verify Data in Postgres

  # Connect to DB
  psql postgresql://octopod:octopod@localhost:5432/octopod_db

  -- GitHub
  SELECT COUNT(*) FROM gh_users;
  SELECT login, followers, public_repos FROM gh_users ORDER BY followers DESC LIMIT 10;
  SELECT COUNT(*) FROM gh_repositories;
  SELECT COUNT(*) FROM gh_commits;
  SELECT status, COUNT(*) FROM gh_checkpoints GROUP BY status;

  -- HuggingFace
  SELECT COUNT(*) FROM hf_users;
  SELECT username, num_models, num_followers FROM hf_users ORDER BY num_followers DESC LIMIT 10;
  SELECT COUNT(*) FROM hf_models;
  SELECT COUNT(*) FROM hf_datasets;
  SELECT status, COUNT(*) FROM hf_checkpoints GROUP BY status;

  6. Test Retry (for any failures)

  poetry run python -m app.ingest.cli gh-retry --status failed --max-attempts 3
  poetry run python -m app.ingest.cli hf-retry --status failed --max-attempts 3

  7. Via API (with dev server running)

  make dev

  # In another terminal:
  curl -X POST http://localhost:8000/api/v1/ingest/gh/run \
    -H "Content-Type: application/json" \
    -d '{"logins": ["torvalds", "gvanrossum"]}'

  curl http://localhost:8000/api/v1/ingest/status

  make test  # runs all 131 tests
