│                                                                                                                                                                                                                                                                                                                                                                                                                                │
│ Context                                                                                                                                                                                                                                                                                                                                                                                                                        │
│                                                                                                                                                                                                                                                                                                                                                                                                                                │
│ The ingestion engine works (GitHub + HuggingFace) but jobs are tracked in-memory via a Python dict — lost on server restart, no per-item tracking, no audit trail. We need persistent DB-backed job tracking that:                                                                                                                                                                                                             │
│                                                                                                                                                                                                                                                                                                                                                                                                                                │
│ 1. Survives restarts                                                                                                                                                                                                                                                                                                                                                                                                           │
│ 2. Tracks per-item progress (which user, what was written, errors)                                                                                                                                                                                                                                                                                                                                                             │
│ 3. Links checkpoints to the job that processed them                                                                                                                                                                                                                                                                                                                                                                            │
│ 4. Is forward-compatible with the future workflow execution system (workflow_execution → execution_phase → execution_log)                                                                                                                                                                                                                                                                                                      │

│ Context                                                                                                                                                                                                                                                                                                                                                                                                                        │
│                                                                                                                                                                                                                                                                                                                                                                                                                                │
│ Octopod has two disconnected ingestion layers: (1) bulk ingestion (app/ingest/gh/, app/ingest/hf/) that stores raw data into gh_*/hf_* tables via asyncpg, and (2) a profile system (developer_profile → aggregated_individual_profile → cohesive_individual_profile → Qdrant) that works one-at-a-time via API. These layers are not connected — bulk-ingested data never becomes searchable.                                 │
│                                                                                                                                                                                                                                                                                                                                                                                                                                │
│ This plan bridges them into a unified, trackable pipeline, adds LinkedIn ingestion, adds Amazon OpenSearch alongside Qdrant for search, and adds full merge audit tracking.                                                                                                                                                                                                                                                    │
│                                                                                                                                                                                                                                                                                                                                                                                                                                │
│ Key Design Decisions                                                                                                                                                                                                                                                                                                                                                                                                           │
│                                                                                                                                                                                                                                                                                                                                                                                                                                │
│ 1. Two domain-level merge tables — developer_profile (merged GH+HF data) and social_profile (merged LN+X data)                                                                                                                                                                                                                                                                                                                 │
│ 2. Raw tables untouched — gh_users, hf_users, ln_users stay as source of truth for audit                                                                                                                                                                                                                                                                                                                                       │

Full field-level merge audit — merge_audit_log tracks every merge decision at both domain and final merge levels                                                                                                                                                                                                                                                                                                            │

│ ECS Task Definition                                                                                                                                                                                                                                                                                                                                                                                                            │
│                                                                                                                                                                                                                                                                                                                                                                                                                                │
│ - Fargate, 1 vCPU / 2 GB RAM                                                                                                                                                                                                                                                                                                                                                                                                   │
│ - Same Docker image as API service                                                                                                                                                                                                                                                                                                                                                                                             │
│ - Each step = a CLI command override (e.g., python -m app.ingest.cli gh-discover)                                                                                                                                                                                                                                                                                                                                              │
│ - Secrets from AWS Secrets Manager (GITHUB_TOKENS, PROXYCURL_API_KEY)                                                                                                                                                                                                                                                                                                                                                          │
│ - CloudWatch Logs for audit trail                                                                                                                                                                                                                                                                                                                                                                                              │
│ - Step Functions handles retries, timeouts, error routing between steps                                                                                                                                                                                                                                                                                                                                                        │
│                                                                                                                                                                                                                                                                                                                                                                                                                                │
│ Why ECS + Step Functions (not AWS Glue)                                                                                                                                                                                                                                                                                                                                                                                        │
│                                                                                                                                                                                                                                                                                                                                                                                                                                │
│ - Scale: Starting at 10K-100K profiles. Glue is for Spark-level parallelism at 100M+.                                                                                                                                                                                                                                                                                                                                          │
│ - Cost: ECS Fargate ~$0.04/hr vs Glue minimum ~$0.44/hr (10x cheaper for our volume).                                                                                                                                                                                                                                                                                                                                          │
│ - Dev speed: Python asyncpg code testable locally. Glue requires AWS deployment to test.                                                                                                                                                                                                                                                                                                                                       │
│ - Migration path: If we outgrow ECS, swap individual Step Functions steps to Glue jobs — merge logic stays the same.                                                                                                                                                                                                                                                                                                           │
